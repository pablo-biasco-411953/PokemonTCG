Digital Battle Engine

Simulador Estratégico de TCG basado en Spring Boot y Angular

Descripción del Proyecto

Este sistema implementa un motor de reglas complejo para el juego de cartas coleccionables Pokémon TCG. A diferencia de una aplicación estática, el proyecto gestiona una máquina de estados de batalla en tiempo real, validación de recursos y una Inteligencia Artificial estratégica.

Arquitectura Técnica

🧠 Backend (Spring Boot)

  • Java 21: Uso de Virtual Threads y Pattern Matching.
  • H2 Database: Persistencia ágil en memoria para testing.
  • Jackson: Hidratación de JSON para modelos polimórficos.
  • Async Service: Procesamiento asíncrono para el turno de la IA.

🎨 Frontend (Angular)

  • Angular 17: Componentes reactivos y Standalone API.
  • RxJS: Gestión de estado mediante observables.
  • Tailwind CSS: Layout dinámico para el tablero de juego.

Inteligencia Artificial Estratégica

El BotAIService no realiza movimientos aleatorios. Utiliza un Sistema de Evaluación Heurística para puntuar cada carta en mano y banca antes de actuar:

// Fragmento de la lógica de decisión de la IA
int score = (energiaUnida * 50) + (hpActual * 1);
if (esDebilAlRival) score -= 1000;  // Evita el K.O. técnico
if (esFuerteContraRival) score += 500; // Prioriza la ventaja elemental

🛡️ Retirada Táctica

Si el activo del Bot está a punto de ser derrotado o no tiene energías para atacar, la IA paga el costo de retirada y sube un "tanque" de la banca.

⚔️ Análisis de Matchup

Evaluación constante de Weakness (x2) y Resistance (-20) para determinar la efectividad del ataque.

Gestión de Datos y Colección

El sistema cuenta con un DataLoader automático que sincroniza datos de la API oficial con el entorno local del desarrollador.

Entidad Estado Funcionalidad
Carga de Cartas Listo Importación de 255 cartas (Pokémon, Trainers, Energy).
Manejo de Sobres Listo Generación de boosters con ratio 30/70 energía/pokes.
Mazo Automático Listo Creación de mazo legal de 60 cartas para el usuario "Pablo".
Estados Alterados En Desarrollo Implementación de Confused, Paralyzed y Poisoned.

API REST - Endpoints Principales

POST /api/battle/start/{user}     // Inicializa el tablero y mezcla mazos
POST /api/battle/{id}/attach      // Une energía de la mano a un Pokémon
POST /api/battle/{id}/attack      // Calcula daño real considerando debilidades
GET  /api/battle/{id}/state       // Sincroniza el estado visual del tablero
